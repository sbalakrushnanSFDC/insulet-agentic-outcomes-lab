# Insulet Agentic Outcomes Lab

> A curated gallery of AI-generated workflow outputs for structured human-in-the-loop (HITL) evaluation.

---

## Purpose

This repository hosts the **outputs** produced by the Insulet agentic implementation pipeline — not the code that runs it. Every artifact here was generated by an AI orchestrator working through a disciplined, 10-step Salesforce SDLC pipeline, then curated by a human operator before publication.

The goal is to give cross-functional reviewers a safe, structured space to:

- **Inspect** what the agentic workflow produced for a given Jira feature
- **Validate** whether the outputs are useful, complete, and accurate
- **Identify gaps** in the agent's reasoning, prompting, or data sources
- **Drive improvement** back into the orchestration pipeline through structured feedback

Partial outputs and agentic drafts are **expected and welcome**. They surface where the pipeline needs refinement more than polished outputs ever could.

## How to Navigate

```
outcomes/
  {YEAR-MONTH}/
    {JIRA-FEATURE-ID}/
      outcome-metadata.md    <- start here: source refs, assumptions, known gaps
      pipeline/              <- step-by-step agent handoff artifacts (00-intake → 09-docs)
      solution/              <- synthesized architecture and design documents
```

1. Browse to `outcomes/` and select a month.
2. Open a feature folder (e.g. `NGOMCT-344`).
3. Start with `outcome-metadata.md` — it tells you the Jira source, what the agent assumed, and what gaps were identified before publishing.
4. Then review the artifacts in `pipeline/` in order (00 → 09).
5. Read the synthesized document in `solution/` for the full architecture view.

For a detailed explanation of each artifact type, see [docs/interpretation-guide.md](docs/interpretation-guide.md).

## How to Give Feedback

1. Open a [GitHub Issue](../../issues/new/choose) using the **Outcome Review Feedback** template.
2. Select the feature, artifact, category, and severity.
3. Describe the gap or success.
4. The workflow designer will triage and create a new pipeline iteration.

For governance details, see [GOVERNANCE.md](GOVERNANCE.md).
