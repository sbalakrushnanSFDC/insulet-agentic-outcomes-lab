# NGASIM-556: E2E VALIDATION

## Decisions Made

1. E2E validation for this story is primarily manual (Salesforce Mobile App on iPad). No Playwright/Selenium automation is practical for native mobile app testing.
2. SOQL-based automated checks supplement manual testing for permission and configuration verification.
3. Deploy target: `orgsyncng` sandbox only (per safety constraints -- never DevInt2).

### Automated SOQL Checks (executable against orgsyncng post-deploy)

```sql
-- SQ-01: PSG exists
SELECT Id, DeveloperName, Status
FROM PermissionSetGroup
WHERE DeveloperName = 'Field_Sales_Mobile_PSG'

-- SQ-02: PSG components
SELECT PermissionSet.Name
FROM PermissionSetGroupComponent
WHERE PermissionSetGroup.DeveloperName = 'Field_Sales_Mobile_PSG'

-- SQ-03: Assignment check
SELECT Assignee.Name
FROM PermissionSetAssignment
WHERE PermissionSetGroup.DeveloperName = 'Field_Sales_Mobile_PSG'
LIMIT 5
```
