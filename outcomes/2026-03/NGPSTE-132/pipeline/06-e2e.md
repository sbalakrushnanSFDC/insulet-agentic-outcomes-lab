# NGPSTE-132: E2E VALIDATION

## Decisions Made

1. E2E validation combines SOQL-based automated checks with manual CTI interaction testing.
2. Live Amazon Connect call tests cannot execute until a sandbox Connect instance is provisioned — these are blocked.
3. Deploy target: `orgsyncng` sandbox only (per safety constraints — never DevInt2).
4. TTEC agent profiles are not available in orgsyncng — tests use a proxy agent user with equivalent permissions.

### Automated SOQL Checks (executable against orgsyncng post-deploy)

```sql
-- SQ-01: ServicePresenceStatus records exist
SELECT Id, DeveloperName, MasterLabel
FROM ServicePresenceStatus
WHERE DeveloperName IN ('Available_Voice', 'Busy_Voice', 'After_Call_Work')

-- SQ-02: PSG contains expected permission sets
SELECT PermissionSet.Name
FROM PermissionSetGroupComponent
WHERE PermissionSetGroup.DeveloperName = 'CTI_Agent_Access'

-- SQ-03: CallCenter adapter verification
SELECT Id, Name, InternalName
FROM CallCenter
WHERE InternalName = 'ACLightningAdapter'

-- SQ-04: VoiceCall records post-test (run after live call test)
SELECT Id, CallType, CallStartDateTime, CallEndDateTime, CallDurationInSeconds
FROM VoiceCall
WHERE CreatedDate = TODAY
ORDER BY CallStartDateTime DESC
LIMIT 10

-- SQ-05: ServiceChannel verification
SELECT Id, DeveloperName, RelatedEntity
FROM ServiceChannel
WHERE DeveloperName = 'sfdc_phone'

-- SQ-06: PSG assignment check
SELECT Assignee.Name, PermissionSetGroup.DeveloperName
FROM PermissionSetAssignment
WHERE PermissionSetGroup.DeveloperName = 'CTI_Agent_Access'
LIMIT 10
```
